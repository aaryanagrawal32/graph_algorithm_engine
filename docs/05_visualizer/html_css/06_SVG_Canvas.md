# 06. The Main Canvas and SVG Physics
*Reference: `visualizer/index.html` (Lines 101 - 158) and `visualizer/style.css` (Lines 473 - 718)*

This document breaks down the HTML structure of the right-side canvas and explains the incredibly powerful CSS rules that animate the SVG graphics generated by the physics engine.

---

## 1. The HTML Canvas (`index.html`)

```html
        <!-- Main Visualization Canvas -->
        <main class="main-content">
            <!-- SVG Canvas for D3.js -->
            <svg id="graph-svg" width="100%" height="100%"></svg>
```
**Explanation (Lines 101 - 104):**
- `<main class="main-content">` is the wrapper for the entire right side of the screen.
- `<svg>` is the Scalable Vector Graphics tag. Unlike a `<canvas>` which is drawn with pixels, SVG shapes are drawn using math. This is what allows D3.js to zoom in 400% without the circles getting blurry!
- `width="100%" height="100%"` ensures the SVG stretches to perfectly fit the `main-content` wrapper.

## 3. Overlay Modal (`index.html` & `style.css`)

```html
            <!-- Instructions Overlay (disappears when graph loads) -->
            <div id="instructions-overlay" class="overlay-modal">
                <div class="modal-content">
                    <h2>Ready to Visualize</h2>
                    <p>Load a graph from the sidebar to begin.</p>
                </div>
            </div>
        </main>
```
**Explanation (Lines 116 - 158):**
- This creates the large text box floating in the middle of the empty canvas.

```css
.overlay-modal {
    position: absolute;
    top: 50%;
    left: 50%;
    transform: translate(-50%, -50%);
    text-align: center;
    pointer-events: none; /* Let clicks pass through to the canvas */
    transition: opacity 0.5s ease;
}
```
**Explanation (Lines 522 - 530):**
- **The Classic Centering Trick:** To center an absolutely positioned box, you move its top-left corner to the dead center (`top: 50%; left: 50%;`), and then use `transform: translate(-50%, -50%)` to shift the box back and up by exactly half of its own width and height, perfectly centering it!
- `pointer-events: none;` means if the user tries to click the text, the mouse click goes *through* the text and hits the SVG canvas behind it (so they can still drag and drop files).

## 4. D3 Physics SVG Elements (`style.css`)

*Note: You won't find these tags in HTML! These are the tags that D3.js creates dynamically in memory using the physics engine.*

### Links (Edges)
```css
/* ============================================================================
   SVG GRAPH ELEMENT STYLES (D3.js)
   
   Note: We don't style these elements using JavaScript. JavaScript just binds 
   the data and gives them CSS classes. These rules instantly style them!
   ============================================================================ */
/* Links (Edges) */
.graph-link {
    stroke: var(--edge-stroke);
    stroke-opacity: 0.6;
    transition: stroke 0.3s ease, stroke-width 0.3s ease, stroke-opacity 0.3s ease;
}
```
**Explanation (Lines 599 - 611):**
- `stroke` is the SVG equivalent of `border-color`. It paints the actual line dark gray.
- `transition: ...` guarantees that if JavaScript later adds a class that changes the color or width of this line, it will animate smoothly over 0.3 seconds rather than instantly snapping.

### Nodes (Vertices)
```css
/* Nodes */
.node-circle {
    fill: #1E293B;
    stroke: #475569;
    stroke-width: 2px;
    transition: all 0.3s ease;
}

.node-group:hover .node-circle {
    stroke: var(--primary);
    filter: drop-shadow(0 0 8px var(--primary-glow));
    cursor: grab;
}
```
**Explanation (Lines 613 - 624):**
- `fill` is the SVG equivalent of `background-color`. It paints the inside of the circle dark slate.
- `.node-group:hover .node-circle` is an advanced selector. It says: "If the user hovers over the entire `<g>` folder (which includes the circle and the text), then change the stroke of the `<circle>` inside it to blue and add a glowing drop shadow!"

## 5. Animation Classes (The Magic)

*These are the classes that JavaScript's rendering loop violently applies and removes during the animation playback.*

```css
/* --- ALGORITHM STATES --- */

/* BFS/DFS States */
.node-circle.visited {
    fill: var(--primary);
    stroke: var(--primary-glow);
}

.node-circle.current {
    fill: var(--accent);
    stroke: var(--text-main);
    filter: drop-shadow(0 0 12px var(--accent));
}
```
**Explanation (Lines 636 - 647):**
- When JavaScript applies `.visited`, the circle instantly turns dark blue (`fill`).
- When JavaScript applies `.current`, the circle instantly turns hot pink (`fill`), receives a bright white border (`stroke`), and casts an intense pink shadow (`filter`).

```css
/* Shortest Path States */
.node-circle.path-node {
    fill: var(--accent);
    stroke: #fff;
    stroke-width: 3px;
    filter: drop-shadow(0 0 12px var(--accent));
}

.graph-link.path-edge {
    stroke: var(--accent);
    stroke-width: 3px;
    stroke-opacity: 1;
    filter: drop-shadow(0 0 8px var(--accent));
}
```
**Explanation (Lines 657 - 669):**
- Similar to `.current`, these classes define the thick, hot pink glowing aesthetic for Dijkstra and A* shortest paths. The `path-edge` targets the lines connecting the nodes.

```css
/* Kruskal MST States */
.node-circle.mst-node {
    fill: var(--success);
    stroke: #fff;
    filter: drop-shadow(0 0 12px var(--success));
}

.graph-link.mst-edge {
    stroke: var(--success);
    stroke-width: 3px;
    stroke-opacity: 1;
    filter: drop-shadow(0 0 8px var(--success));
}
```
**Explanation (Lines 671 - 683):**
- These classes define the thick, green glowing aesthetic for Kruskal's Minimum Spanning Tree.
- Because JavaScript just applies these classes, we can instantly switch the entire app's visual style from Pink to Green without touching a single line of JavaScript code!
