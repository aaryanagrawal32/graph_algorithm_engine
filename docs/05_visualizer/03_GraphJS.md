## ✦ High-Level Overview: What does `graph.js` do?

At a high level, `graph.js` does three things:
1. **Reads files:** It takes the JSON files generated by your C++ program and parses them into memory.
2. **Draws the Graph:** It uses a physics engine to calculate where nodes should go on the screen so they don't overlap, and draws them as SVG circles and lines.
3. **Runs Animations:** It acts like a movie player. It reads the algorithm steps, increments a "frame counter" every half-second, and changes the colors of the circles to match that frame.

---

## Part 0: The Core Building Blocks

### What is D3.js?
D3 (Data-Driven Documents) is the JavaScript library we use to draw the graph. 
In C++ SFML, you use a `while(window.isOpen())` loop and manually call `window.draw(circle)` every single frame.

**D3 uses "Data Binding" instead of manual drawing.**
You give D3 a list of data (e.g., `[{id: 1}, {id: 2}]`). D3 creates an HTML element for each item in the list. If you update the data, D3 automatically updates the HTML elements for you.

### What is SVG?
SVG (Scalable Vector Graphics) is an HTML tag `<svg>` that acts as a blank canvas. Inside it, you can place mathematical shapes:
- `<circle cx="10" cy="10" r="5"></circle>` draws a circle.
- `<line x1="0" y1="0" x2="10" y2="10"></line>` draws a line.

Our entire graph is just a collection of `<circle>` and `<line>` tags inside the `<svg>` canvas.

## Part 1: Section-by-Section Walkthrough

### 1. Global State Variables (Lines 5 - 43)

```javascript
let graphData = null;       // Holds { nodes: [...], links: [...] }
let algorithmData = {
    bfs: null,             // { steps: [...] }
    dfs: null,             // { steps: [...] }
    dijkstra: null,        // { path: [...] }
    // ...
};

let activeAlgorithm = "";
let currentStep = 0;
let maxSteps = 0;
let isPlaying = false;
```
These are the application's global state. 
- `graphData` stores the physical graph structure (the vertices and edges).
- `algorithmData` stores the JSON traces generated by C++. 
- `currentStep` acts as the "playhead" in our movie. If a BFS took 10 steps, and `currentStep` is 3, we only highlight the first 3 nodes.

### 2. Initialization and Event Listeners (Lines 48 - 98)

```javascript
document.addEventListener("DOMContentLoaded", () => {
    setupUI();
    setupSVG();
    setupDragAndDrop();
    tryAutoLoadFromServer();
});
```
This is the `main()` equivalent for the web page. We wait for the browser to finish drawing the `index.html` structure (`DOMContentLoaded`). Once the HTML is ready, we set up our event listeners.

```javascript
function setupUI() {
    document.getElementById("btn-play").addEventListener("click", togglePlay);
    document.getElementById("btn-next").addEventListener("click", stepForward);
    
    // ...
}
```
**`addEventListener`** is how JS handles user input. Instead of writing a massive C++ `switch` statement inside a while loop to check for clicks, JS is event-driven. You tell the browser: "When the user clicks the element with ID `btn-play`, run the `togglePlay` function."

### 3. Reading the JSON Files (Lines 160 - 202)

Because this is a web app running locally without a backend server, we can't easily use C++ file loading techniques (like `std::ifstream`). Browsers strictly block reading local hard drive files for security reasons. 

To get around this, we use HTML Drag-and-Drop combined with the `FileReader` API:

```javascript
function handleFiles(files) {
    for (let i = 0; i < files.length; ++i) {
        const file = files[i];
        const reader = new FileReader();

        reader.onload = (event) => {
            const data = JSON.parse(event.target.result);
            processJSONData(file.name.toLowerCase(), data);
        };
        reader.readAsText(file); // Starts the asynchronous read
    }
}
```
When you drop a file on the drop-zone, we create a `FileReader`. 
We tell it: "When you finish loading (`onload`), run this arrow function." Then we command it to `readAsText()`. 
`JSON.parse()` magically takes the raw string data and turns it into a fully structured JavaScript object!

### 4. The D3 Physics Engine (Lines 312 - 356)

When we load `graph.json`, it has nodes and links, but **it doesn't have X/Y coordinates**. How do we know where to draw them on the screen? We use a force-directed layout (physics engine).

```javascript
simulation = d3.forceSimulation(graphData.nodes)
    .force("link", d3.forceLink(graphData.links).id(d => d.id).distance(120))
    .force("charge", d3.forceManyBody().strength(-350))
    .force("collide", d3.forceCollide().radius(25))
    .force("center", d3.forceCenter(width / 2, height / 2));
```
This is the core of the visualizer.
1. `d3.forceSimulation` gives every node an X and Y coordinate, starting them randomly near the center.
2. `forceLink` acts like **springs**. Every edge tries to pull its two nodes exactly 120 pixels apart.
3. `forceManyBody` acts like **magnets**. A negative strength (-350) means every node repels every other node, preventing them from clumping together.
4. `forceCollide` adds a physical hitbox (radius 25) so nodes can never overlap.
5. `forceCenter` pulls the entire cluster into the middle of your screen so it doesn't float away.

### 5. Drawing the Nodes and Edges (Lines 358 - 396)

Now that the physics engine is running, we need to actually draw circles and lines.

```javascript
linkSelection = linkG.selectAll(".graph-link")
    .data(graphData.links)
    .enter().append("line")
    .attr("class", "graph-link")
    .attr("stroke-width", 2);
```
**This is the famous D3 Data-Binding Pattern:**
1. `selectAll(".graph-link")` — Select all lines on the screen (currently there are 0).
2. `.data(graphData.links)` — Bind our array of 50 edges to this selection.
3. `.enter()` — We have 0 lines but 50 edges. So, loop 50 times for the "entering" data.
4. `.append("line")` — Create an actual SVG `<line>` tag for each edge.
5. `.attr(...)` — Set the class and thickness of those new lines.

### 6. The Animation Loop: `simulation.on("tick")` (Lines 398 - 411)

The physics engine calculates math, but it doesn't draw anything. It runs at 60 frames per second, and on every frame, it fires a `"tick"` event.

```javascript
simulation.on("tick", () => {
    linkSelection
        .attr("x1", d => d.source.x)
        .attr("y1", d => d.source.y)
        .attr("x2", d => d.target.x)
        .attr("y2", d => d.target.y);

    nodeSelection.attr("transform", d => `translate(${d.x}, ${d.y})`);
});
```
Every frame, we grab all the lines (`linkSelection`). We take the mathematical coordinates calculated by the physics engine (`d.source.x`) and apply them to the physical HTML attributes (`x1`, `y1`).
This is what makes the graph wiggle and settle into place smoothly when you first load it!

### 7. The Playback Engine (Lines 580 - 646)

How do the algorithm animations work? It's just a simple timer.

```javascript
let playbackInterval = null;

function play() {
    isPlaying = true;
    playbackInterval = setInterval(() => {
        if (currentStep < maxSteps) {
            stepForward();
        } else {
            pause(); // We reached the end
        }
    }, playbackSpeed); // e.g., 500 milliseconds
}
```
`setInterval(function, time)` tells the browser: "Run this function every 500ms forever until I tell you to stop."
Every 500ms, it calls `stepForward()`, which adds 1 to `currentStep` and calls `updateSimulationFrame()`.

### 8. Rendering the Algorithm Colors (Lines 670 - 792)

Here is where the magic happens. How do we make the shortest path turn pink?

```javascript
function renderPathfindingFrame() {
    const path = algorithmData[activeAlgorithm].path; // e.g., ["0", "1", "4", "5"]
    
    // Slice the array up to our current playhead frame
    const pathNodes = new Set(path.slice(0, currentStep));

    // Loop through ALL nodes on the screen
    svg.selectAll(".node-circle")
        .classed("path-node", d => pathNodes.has(d.id.toString()));
}
```
**This is the most important concept to understand:**
We **DO NOT** manually change the color in JavaScript. We do not write `node.color = "red"`. 

Instead, we use D3's `.classed("class-name", condition)` function. 
We look at every circle on the screen. We ask: "Is this node's ID inside the `pathNodes` set?"
- If True: D3 adds the HTML attribute `class="path-node"`.
- If False: D3 removes the class.

In `style.css`, we previously wrote a rule that says `.path-node { stroke: pink; filter: drop-shadow(pink); }`. 

Because JavaScript added the class, the browser's CSS engine instantly applies the pink styling and the glowing animation! By separating logic (JS) from styling (CSS), the code is incredibly clean.

## Part 2: Manual Trace of the Playback Engine

Let's trace what happens when you press the Play button on a Dijkstra visualization where the path is `["0", "3", "7"]`.

1. User clicks the ▶ button. `togglePlay()` is fired.
2. `play()` is called. `currentStep` is 0.
3. `setInterval` starts a timer for 500ms.
4. **Timer fires (0.5s):** 
   - `stepForward()` runs. `currentStep` becomes 1.
   - `renderPathfindingFrame()` runs.
   - `path.slice(0, 1)` yields `["0"]`.
   - D3 checks all nodes. Node 0 gets the `path-node` class. Node 0 turns pink.
5. **Timer fires (1.0s):**
   - `stepForward()` runs. `currentStep` becomes 2.
   - `path.slice(0, 2)` yields `["0", "3"]`.
   - D3 checks all nodes. Nodes 0 and 3 get the `path-node` class. Node 3 turns pink.
6. **Timer fires (1.5s):**
   - `stepForward()` runs. `currentStep` becomes 3.
   - `path.slice(0, 3)` yields `["0", "3", "7"]`.
   - D3 checks all nodes. Nodes 0, 3, and 7 turn pink.
   - `currentStep == maxSteps`, so the animation pauses.

## Summary Reference Table

| JavaScript Concept / Function | Plain English Meaning |
|---|---|
| `document.addEventListener` | Wait for the user to do something (click, type, load page) |
| `JSON.parse(string)` | Takes text from a file and turns it into readable data |
| `setInterval(func, time)` | A timer that runs a function repeatedly (our animation loop) |
| `d3.forceSimulation()` | Starts the physics engine that calculates graph layouts |
| `d3.forceManyBody()` | Adds negative gravity (repulsion) so nodes don't overlap |
| `.selectAll().data().enter()` | The D3 formula for creating an HTML tag for every item in an array |
| `simulation.on("tick")` | A loop that runs 60 times a second to update screen graphics |
| `svg.selectAll().classed(cls, bool)` | Adds or removes a CSS class from an element based on a true/false condition |
